# Unit 2: Frontend Customer - 비즈니스 규칙

---

## BR-FC01: 자동 로그인

| 항목 | 규칙 |
|------|------|
| **조건** | 앱 시작 시 localStorage에 인증 정보(token, storeId, tableId)가 존재 |
| **동작** | 저장된 토큰으로 API 요청 테스트 → 유효하면 `/menu`로 이동 |
| **실패 시** | localStorage 클리어 → `/` (설정 화면)으로 이동 |
| **토큰 만료** | 24시간 만료, 401 응답 시 자동 로그아웃 |

---

## BR-FC02: 테이블 로그인

| 항목 | 규칙 |
|------|------|
| **필수 입력** | 매장 식별자(storeCode), 테이블 번호(tableNumber), 비밀번호(password) |
| **유효성** | 모든 필드 비어있지 않을 것, 테이블 번호는 양의 정수 |
| **성공** | 응답의 token, storeId, tableId, storeName을 authStore에 저장 |
| **실패** | "매장 정보를 확인해주세요" 또는 "비밀번호가 올바르지 않습니다" notification 표시 |

---

## BR-FC03: 카테고리 탭 네비게이션

| 항목 | 규칙 |
|------|------|
| **기본 선택** | 첫 번째 카테고리 (displayOrder 기준 정렬, "전체" 탭 없음) |
| **전환** | 클라이언트 사이드 필터링 (전체 메뉴를 1회 로드 후 카테고리별 필터) |
| **정렬** | 카테고리: displayOrder 오름차순, 메뉴: displayOrder 오름차순 |

---

## BR-FC04: 메뉴 카드 상호작용

| 항목 | 규칙 |
|------|------|
| **클릭 동작** | 장바구니에 해당 메뉴 1개 즉시 추가 |
| **중복 추가** | 이미 장바구니에 있으면 수량 +1 (최대 10개까지) |
| **품절 메뉴** | isAvailable=false일 때 "품절" 오버레이, 클릭 비활성화 |
| **가격 표시** | 천 단위 콤마 형식 (예: 9,000원) |
| **이미지** | imageUrl이 null/empty면 기본 placeholder 이미지 표시 |
| **추가 피드백** | notification으로 "장바구니에 추가되었습니다" 표시 |
| **수량 초과** | 이미 10개일 때 추가 시도 → notification "최대 수량(10개)에 도달했습니다" |

---

## BR-FC05: 장바구니 관리

| 항목 | 규칙 |
|------|------|
| **수량 범위** | 최소 1개, 최대 10개 |
| **수량 감소** | 1개에서 `-` 클릭 시 해당 항목 삭제 (확인 없이) |
| **삭제** | 삭제 버튼 클릭 시 즉시 제거 (확인 없이) |
| **영속성** | localStorage에 자동 저장 (Zustand persist 미들웨어) |
| **새로고침** | 페이지 새로고침 시 장바구니 유지 |
| **총 금액** | 실시간 재계산: SUM(각 항목의 price * quantity) |
| **빈 장바구니** | 빈 상태 일러스트 + "메뉴 보러가기" 버튼 |
| **주문하기 버튼** | 장바구니 비어있으면 비활성화 |

---

## BR-FC06: 주문 생성

| 항목 | 규칙 |
|------|------|
| **전제조건** | 장바구니에 1개 이상 항목 |
| **요청 데이터** | `{ items: [{ menuId, quantity }] }` |
| **이중 클릭 방지** | 주문 전송 중 버튼 비활성화 + 로딩 상태 표시 |
| **성공** | 장바구니 클리어 → 주문 결과 화면 → 5초 카운트다운 → `/menu` 리다이렉트 |
| **실패** | notification 에러 표시, 장바구니 유지 |
| **품절 에러** | "주문할 수 없는 메뉴가 포함되어 있습니다" → 장바구니로 돌아가기 |

---

## BR-FC07: 주문 내역 조회

| 항목 | 규칙 |
|------|------|
| **범위** | 현재 활성 세션의 주문만 표시 |
| **정렬** | 최신 주문이 위 (createdAt 역순) |
| **갱신** | 수동 새로고침 버튼 클릭 시 재조회 |
| **빈 상태** | "아직 주문 내역이 없습니다" 안내 메시지 |
| **상태 표시** | OrderStatusBadge 컴포넌트 (WAITING=주황, PREPARING=파랑, COMPLETE=초록) |
| **총 금액** | 모든 주문의 totalAmount 합산 표시 |

---

## BR-FC08: 에러/알림 표시

| 항목 | 규칙 |
|------|------|
| **방식** | Ant Design notification (화면 우측 상단) |
| **성공** | type: success, 3초 자동 닫힘 |
| **에러** | type: error, 5초 자동 닫힘 |
| **네트워크 에러** | "서버와 연결할 수 없습니다. 잠시 후 다시 시도해주세요." |
| **401 에러** | 자동 로그아웃 + "/" 리다이렉트 + "인증이 만료되었습니다" |

---

## BR-FC09: Axios 인터셉터

| 항목 | 규칙 |
|------|------|
| **Request** | authStore.token이 있으면 `Authorization: Bearer {token}` 헤더 추가 |
| **Response 401** | authStore.logout() → `/` 리다이렉트 → notification "인증이 만료되었습니다" |
| **Response 4xx** | 서버 에러 메시지 추출하여 notification 표시 |
| **Response 5xx** | "서버 오류가 발생했습니다" notification |
| **Timeout** | 10초, "요청 시간이 초과되었습니다" notification |

## BR-FC10: 라우팅 가드

| 항목 | 규칙 |
|------|------|
| **테이블 인증 필요** | `/menu`, `/cart`, `/order-confirm`, `/orders` |
| **미인증 접근** | `/`로 리다이렉트 |
| **인증 상태에서 `/` 접근** | `/menu`로 리다이렉트 |
| **인증 확인** | authStore.isAuthenticated && authStore.role === 'TABLE' |
