# 테이블오더 서비스 - 사용자 스토리

> **분해 방식**: User Journey-Based (사용자 여정 흐름 중심)
> **수락 기준**: Given/When/Then 간결 형식 (스토리당 2~3개)
> **우선순위**: 사용자 플로우 순서

---

## Journey 1: 고객 주문 플로우

### US-C01: 테이블 태블릿 초기 설정
**As a** 관리자,
**I want to** 태블릿에 매장/테이블 정보를 1회 설정하면 자동 로그인되도록,
**So that** 고객이 별도 로그인 없이 즉시 주문할 수 있다.

**Acceptance Criteria:**
- **Given** 태블릿에 설정 정보가 없을 때, **When** 관리자가 매장 식별자, 테이블 번호, 비밀번호를 입력하면, **Then** 정보가 로컬에 저장되고 자동 로그인된다.
- **Given** 태블릿에 설정 정보가 저장되어 있을 때, **When** 페이지를 새로고침하면, **Then** 자동 로그인되어 메뉴 화면이 표시된다.
- **Given** 저장된 인증 정보가 유효하지 않을 때, **When** 자동 로그인을 시도하면, **Then** 에러 메시지와 함께 설정 화면이 표시된다.

**Persona**: 관리자(박서연) → 고객(김민수)

---

### US-C02: 메뉴 조회 및 탐색
**As a** 고객,
**I want to** 카테고리별로 메뉴를 사진과 함께 살펴볼 수 있도록,
**So that** 원하는 메뉴를 쉽게 찾아 선택할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인된 상태에서, **When** 메뉴 화면에 접속하면, **Then** 카테고리 탭과 해당 카테고리의 메뉴 카드(이름, 가격, 설명, 이미지)가 표시된다.
- **Given** 메뉴 화면에서, **When** 다른 카테고리 탭을 선택하면, **Then** 해당 카테고리의 메뉴 목록으로 즉시 전환된다.

**Persona**: 고객(김민수)

---

### US-C03: 장바구니에 메뉴 추가
**As a** 고객,
**I want to** 원하는 메뉴를 장바구니에 담고 수량을 조절할 수 있도록,
**So that** 주문 전에 선택한 메뉴를 검토하고 수정할 수 있다.

**Acceptance Criteria:**
- **Given** 메뉴 화면에서, **When** 메뉴를 선택하면, **Then** 장바구니에 해당 메뉴가 추가되고 총 금액이 갱신된다.
- **Given** 장바구니에 메뉴가 있을 때, **When** 수량을 증가/감소하면, **Then** 수량과 총 금액이 실시간으로 재계산된다.
- **Given** 장바구니에 메뉴가 있을 때, **When** 페이지를 새로고침하면, **Then** 장바구니 내용이 유지된다 (로컬 저장).

**Persona**: 고객(김민수)

---

### US-C04: 주문 생성 및 확인
**As a** 고객,
**I want to** 장바구니의 메뉴를 주문으로 확정하고 결과를 확인할 수 있도록,
**So that** 주문이 정상 접수되었음을 알 수 있다.

**Acceptance Criteria:**
- **Given** 장바구니에 메뉴가 있을 때, **When** 주문 확정 버튼을 누르면, **Then** 주문이 서버에 전송되고 주문 번호가 표시된다.
- **Given** 주문이 성공하면, **When** 5초 경과 후, **Then** 메뉴 화면으로 자동 리다이렉트되고 장바구니가 비워진다.
- **Given** 주문이 실패하면, **When** 에러가 발생하면, **Then** 에러 메시지가 표시되고 장바구니는 유지된다.

**Persona**: 고객(김민수)

---

### US-C05: 주문 내역 조회
**As a** 고객,
**I want to** 현재 식사 세션의 주문 내역과 상태를 확인할 수 있도록,
**So that** 주문 진행 상황을 파악할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인된 상태에서, **When** 주문 내역 화면에 접속하면, **Then** 현재 세션의 주문 목록이 시간순으로 표시된다 (주문번호, 시각, 메뉴, 금액, 상태).
- **Given** 이전 세션에 주문이 있었더라도, **When** 이용완료 처리 후 새 세션이 시작되면, **Then** 이전 세션의 주문은 표시되지 않는다.

**Persona**: 고객(김민수)

## Journey 2: 관리자 운영 플로우

### US-A01: 관리자 로그인
**As a** 관리자,
**I want to** 매장 식별자와 비밀번호로 로그인할 수 있도록,
**So that** 매장 관리 시스템에 안전하게 접근할 수 있다.

**Acceptance Criteria:**
- **Given** 로그인 화면에서, **When** 올바른 매장 식별자/사용자명/비밀번호를 입력하면, **Then** JWT 토큰이 발급되고 대시보드로 이동한다.
- **Given** 로그인 상태에서, **When** 16시간이 경과하면, **Then** 자동 로그아웃되어 로그인 화면으로 돌아간다.
- **Given** 로그인 상태에서, **When** 브라우저를 새로고침하면, **Then** 세션이 유지된다.

**Persona**: 관리자(박서연)

---

### US-A02: 실시간 주문 모니터링
**As a** 관리자,
**I want to** 들어오는 주문을 실시간으로 테이블별 대시보드에서 확인할 수 있도록,
**So that** 주문을 놓치지 않고 신속하게 처리할 수 있다.

**Acceptance Criteria:**
- **Given** 대시보드에 접속한 상태에서, **When** 고객이 새 주문을 생성하면, **Then** 2초 이내에 해당 테이블 카드에 주문이 표시되고 시각적으로 강조된다.
- **Given** 대시보드에서 테이블 카드를 확인할 때, **When** 카드를 클릭하면, **Then** 해당 테이블의 전체 주문 메뉴 상세가 표시된다.

**Persona**: 관리자(박서연)

---

### US-A03: 주문 상태 변경
**As a** 관리자,
**I want to** 주문 상태를 대기중에서 준비중, 완료로 변경할 수 있도록,
**So that** 고객과 내부 직원이 주문 진행 상황을 알 수 있다.

**Acceptance Criteria:**
- **Given** 대시보드에서 주문을 확인할 때, **When** 상태 변경 버튼을 누르면, **Then** 주문 상태가 순차적으로 변경된다 (대기중 → 준비중 → 완료).
- **Given** 주문 상태가 변경되면, **When** 고객이 주문 내역을 조회하면, **Then** 변경된 상태가 반영되어 표시된다.

**Persona**: 관리자(박서연)

---

### US-A04: 주문 삭제 (직권 수정)
**As a** 관리자,
**I want to** 잘못된 주문을 삭제할 수 있도록,
**So that** 주문 오류를 즉시 수정하고 정확한 주문액을 유지할 수 있다.

**Acceptance Criteria:**
- **Given** 테이블 상세에서, **When** 주문 삭제 버튼을 누르면, **Then** 확인 팝업이 표시된다.
- **Given** 확인 팝업에서 삭제를 확정하면, **When** 삭제가 완료되면, **Then** 주문이 즉시 제거되고 테이블 총 주문액이 재계산된다.

**Persona**: 관리자(박서연)

---

### US-A05: 테이블 이용완료 처리
**As a** 관리자,
**I want to** 고객 퇴장 시 테이블을 "이용완료" 처리할 수 있도록,
**So that** 새 고객이 이전 주문 없이 깨끗한 상태에서 시작할 수 있다.

**Acceptance Criteria:**
- **Given** 테이블에 활성 세션이 있을 때, **When** 이용완료 버튼을 누르고 확인하면, **Then** 현재 세션이 종료되고 주문 내역이 과거 이력으로 이동된다.
- **Given** 이용완료 처리가 되면, **When** 대시보드를 확인하면, **Then** 해당 테이블의 현재 주문 목록과 총 주문액이 0으로 리셋된다.
- **Given** 이용완료 처리 후, **When** 해당 테이블에서 새 주문이 들어오면, **Then** 새로운 테이블 세션이 자동 시작된다.

**Persona**: 관리자(박서연)

---

### US-A06: 과거 주문 내역 조회
**As a** 관리자,
**I want to** 테이블별 과거 주문 내역을 조회할 수 있도록,
**So that** 이전 고객의 주문 이력을 확인할 수 있다.

**Acceptance Criteria:**
- **Given** 대시보드에서, **When** 테이블의 과거 내역 버튼을 누르면, **Then** 과거 세션의 주문 목록이 시간 역순으로 표시된다 (주문번호, 시각, 메뉴, 금액, 이용완료 시각).
- **Given** 과거 내역 화면에서, **When** 날짜 필터를 적용하면, **Then** 해당 날짜의 주문만 표시된다.

**Persona**: 관리자(박서연)

---

### US-A07: 메뉴 관리
**As a** 관리자,
**I want to** 메뉴를 등록, 수정, 삭제하고 노출 순서를 조정할 수 있도록,
**So that** 매장의 메뉴를 동적으로 관리할 수 있다.

**Acceptance Criteria:**
- **Given** 메뉴 관리 화면에서, **When** 새 메뉴를 등록하면(메뉴명, 가격, 설명, 카테고리, 이미지URL), **Then** 메뉴가 저장되고 고객 화면에 반영된다.
- **Given** 메뉴 관리 화면에서, **When** 기존 메뉴를 수정 또는 삭제하면, **Then** 변경 사항이 즉시 반영된다.
- **Given** 필수 필드가 비어있거나 가격이 유효하지 않을 때, **When** 저장을 시도하면, **Then** 검증 에러 메시지가 표시된다.

**Persona**: 관리자(박서연)
