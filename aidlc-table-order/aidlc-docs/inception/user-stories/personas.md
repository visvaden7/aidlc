# 테이블오더 서비스 - 사용자 페르소나

---

## Persona 1: 고객 (Customer) - "김민수"

### 프로필
- **역할**: 매장 방문 고객
- **사용 기기**: 테이블에 비치된 태블릿 (브라우저 기반)
- **기술 숙련도**: 다양 (스마트폰 사용 가능 수준이면 충분)
- **연령대**: 전 연령층

### 목표
- 직원 호출 없이 빠르고 편하게 주문하고 싶다
- 메뉴를 사진과 설명으로 충분히 살펴보고 선택하고 싶다
- 주문 후 현재 상태를 바로 확인하고 싶다
- 추가 주문을 간편하게 하고 싶다

### 행동 패턴
- 테이블에 앉으면 태블릿 화면에서 바로 메뉴를 탐색
- 여러 메뉴를 장바구니에 담아둔 뒤 한 번에 주문
- 식사 중 추가 주문 발생 가능
- 주문 상태를 수시로 확인

### 불편 요소 (Pain Points)
- 복잡한 로그인 절차
- 메뉴 정보 부족 (사진/설명 없음)
- 주문 후 상태 확인 불가
- 장바구니가 사라지는 경험

---

## Persona 2: 관리자 (Admin) - "박서연"

### 프로필
- **역할**: 매장 운영자/점주
- **사용 기기**: PC 또는 태블릿 (브라우저 기반)
- **기술 숙련도**: 기본적인 웹 사용 가능
- **근무 시간**: 영업 시간 내 상시 접속

### 목표
- 들어오는 주문을 실시간으로 놓치지 않고 확인하고 싶다
- 테이블별 주문 현황을 한눈에 파악하고 싶다
- 메뉴를 쉽게 등록/수정하고 싶다
- 테이블 회전을 효율적으로 관리하고 싶다

### 행동 패턴
- 영업 시작 시 관리자 로그인 후 대시보드 상시 모니터링
- 신규 주문 수신 시 주문 상태를 순차적으로 변경 (대기중 → 준비중 → 완료)
- 고객 퇴장 시 "이용완료" 처리로 테이블 리셋
- 메뉴 변경 시 관리 화면에서 즉시 수정

### 불편 요소 (Pain Points)
- 주문 누락
- 테이블별 현황 파악 어려움
- 잘못된 주문 처리 번거로움
- 이전 고객 주문과 현재 고객 주문 혼동
