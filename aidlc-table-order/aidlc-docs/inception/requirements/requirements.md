## 2. 기능 요구사항 (Functional Requirements)

### 2.1 고객용 기능 (Customer Features)

#### FR-C01: 테이블 태블릿 자동 로그인 및 세션 관리
- **초기 설정 (관리자 1회 수행)**: 매장 식별자, 테이블 번호, 테이블 비밀번호 입력 후 로컬 저장
- **자동 로그인**: 저장된 정보로 자동 인증, 별도 로그인 절차 없이 즉시 주문 가능
- **세션 유지**: 브라우저 새로고침 시에도 로그인 상태 유지

#### FR-C02: 메뉴 조회 및 탐색
- **기본 화면**: 메뉴 화면이 기본 화면으로 항상 표시
- **카테고리 분류**: 사전 정의된 고정 카테고리별 메뉴 분류 및 표시
- **메뉴 상세 정보**: 메뉴명, 가격, 설명, 이미지(외부 URL) 표시
- **카테고리 간 빠른 이동** 지원
- **UI 요구사항**: 카드 형태 레이아웃, 터치 친화적 버튼(최소 44x44px), 명확한 시각적 계층 구조

#### FR-C03: 장바구니 관리
- 메뉴 추가/삭제, 수량 조절(증가/감소)
- 총 금액 실시간 계산
- 장바구니 비우기
- **로컬 저장**: 클라이언트 측 임시 저장 (페이지 새로고침 시에도 유지)
- 서버 전송은 주문 확정 시에만 수행

#### FR-C04: 주문 생성
- 주문 내역 최종 확인
- 주문 확정 버튼
- **주문 성공 플로우**: 주문 번호 표시 → 5초 후 메뉴 화면 자동 리다이렉트 → 장바구니 자동 비우기
- **주문 실패 시**: 에러 메시지 표시, 장바구니 유지
- **주문 정보**: 매장 식별 정보, 테이블 식별 정보, 주문 메뉴 목록(메뉴명, 수량, 단가), 총 주문 금액, 세션 ID

#### FR-C05: 주문 내역 조회
- 주문 시간 순 정렬
- 주문별 상세 정보: 주문 번호, 주문 시각, 주문 메뉴 및 수량, 주문 금액, 주문 상태(대기중/준비중/완료)
- **데이터 필터링**: 현재 테이블 세션 주문만 표시 (이전 식사 세션 제외)
- 매장 이용 완료 처리된 주문 제외

### 2.2 관리자용 기능 (Admin Features)

#### FR-A01: 매장 인증
- 매장 식별자, 사용자명, 비밀번호 입력
- **세션 관리**: JWT 토큰 기반, 16시간 세션 유지, 브라우저 새로고침 시 유지, 16시간 후 자동 로그아웃
- **보안**: 비밀번호 bcrypt 해싱, 로그인 시도 제한

#### FR-A02: 실시간 주문 모니터링
- **SSE(Server-Sent Events)** 기반 실시간 주문 목록 업데이트
- **그리드/대시보드 레이아웃**: 테이블별 카드 형태, 각 카드에 총 주문액 표시, 최신 주문 미리보기
- 주문별 상세 정보: 테이블 번호, 주문 번호, 주문 시각, 주문 메뉴 및 수량(축약), 총 금액
- **주문 카드 클릭 시**: 전체 메뉴 목록 상세 보기
- 주문 상태 변경: 대기중 → 준비중 → 완료
- **신규 주문**: 시각적 강조 (색상 변경, 애니메이션)
- **성능 요건**: 2초 이내 주문 표시

#### FR-A03: 테이블 관리
- **테이블 태블릿 초기 설정**: 테이블 번호 및 비밀번호 설정, 16시간 세션 생성
- **주문 삭제 (직권 수정)**: 삭제 버튼 → 확인 팝업 → 즉시 삭제 → 총 주문액 재계산 → 성공/실패 피드백
- **테이블 세션 처리**:
  - 세션 시작: 첫 주문 시 자동 시작
  - 세션 종료(이용완료): 확인 팝업 → 주문 내역을 과거 이력으로 이동 → 현재 주문 목록 및 총 주문액 리셋 → 새 고객은 이전 내역 없이 시작
- **과거 주문 내역 조회**: 과거 내역 버튼, 시간 역순 표시, 주문 정보(주문 번호, 시각, 메뉴 목록, 총 금액, 이용완료 시각), 날짜 필터링, 닫기 버튼으로 대시보드 복귀

#### FR-A04: 메뉴 관리
- 메뉴 조회 (카테고리별)
- 메뉴 등록: 메뉴명, 가격, 설명, 카테고리(고정 목록에서 선택), 이미지 URL
- 메뉴 수정, 메뉴 삭제
- 메뉴 노출 순서 조정
- **데이터 검증**: 필수 필드 검증, 가격 범위 검증

## 3. 비기능 요구사항 (Non-Functional Requirements)

### NFR-01: 성능
- SSE를 통한 실시간 주문 알림: 2초 이내 표시
- 메뉴 조회 응답 시간: 합리적인 수준 (소규모 데이터)

### NFR-02: 사용성
- 터치 친화적 UI (태블릿 기준 최적화)
- 직관적인 카드 기반 레이아웃
- 명확한 시각적 피드백 (성공/실패)

### NFR-03: 안정성
- 장바구니 로컬 저장 (새로고침 시 유지)
- SSE 연결 끊김 시 자동 재연결
- JWT 만료 시 자동 로그아웃 및 재로그인 유도

### NFR-04: 데이터 무결성
- 주문 항목에 메뉴명/단가 스냅샷 저장 (메뉴 변경 시에도 이력 보존)
- 테이블 세션을 통한 주문 그룹화
- 이용완료 시 과거 이력으로 안전하게 이동

## 4. 제외 기능 (Out of Scope)

다음 기능은 이번 MVP에서 **구현하지 않음**:

| 카테고리 | 제외 항목 |
|----------|-----------|
| 결제 | 실제 결제 처리, PG사 연동, 영수증, 환불, 포인트/쿠폰 |
| 인증 | OAuth, SNS 로그인, 2FA, OTP |
| 파일/컨텐츠 | 이미지 리사이징/최적화, CMS, 광고 |
| 알림 | 푸시 알림, SMS, 이메일, 소리/진동 |
| 주방 | 주문 주방 전달, 재고 관리 |
| 고급 기능 | 데이터 분석, 매출 리포트, 재고 관리, 직원 관리, 예약, 리뷰, 다국어 |
| 외부 연동 | 배달 플랫폼, POS, 소셜 미디어, 지도 API, 번역 API |

## 5. 용어 정의

| 용어 | 정의 |
|------|------|
| **테이블 세션** | 특정 테이블에 고객이 앉아서 첫 주문을 시작한 후부터 해당 테이블 이용완료 처리까지의 기간. 세션 종료 후 다른 고객의 첫 주문 시 새로운 세션 시작. |
| **이용완료** | 관리자가 테이블 세션을 종료하는 행위. 해당 세션의 모든 주문이 과거 이력으로 이동됨. |
| **자동 로그인** | 태블릿에 저장된 매장/테이블 정보로 별도 입력 없이 자동으로 인증되는 기능. |
| **SSE** | Server-Sent Events. 서버에서 클라이언트로 실시간 이벤트를 단방향 전송하는 기술. |
| **시드 데이터** | 개발/테스트용으로 미리 삽입되는 샘플 데이터 (매장, 카테고리, 메뉴, 테이블 등). |
